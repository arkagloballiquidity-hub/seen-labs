# Implementation Plan: Seen Labs Landing Page

**Branch**: `001-seenlabs-landing` | **Date**: 2026-05-30 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `specs/001-seenlabs-landing/spec.md`

---

## Summary

Build a cinematic, single-page dark landing page for Seen Labs ("Out of the Dark") using
React + Vite + TypeScript + Tailwind CSS + GSAP + Framer Motion + hls.js. The site is a
static SPA deployed to Vercel. It features a loading screen with animated counter, HLS video
hero with GSAP entrance animations, Framer Motion scroll reveals for all content sections,
a GSAP marquee footer, and a floating pill navbar — all using the Seen Labs purple brand
system and the copy from the reference `seen-labs.html`.

---

## Technical Context

**Language/Version**: TypeScript 5.x + React 18

**Primary Dependencies**:
- `vite` 5.x — build tool and dev server
- `tailwindcss` 3.x — utility-first CSS with custom brand tokens
- `gsap` 3.x — loading screen animation, hero entrance, marquee, scroll-driven parallax
- `framer-motion` 11.x — section whileInView reveals, AnimatePresence (loading→hero transition, FAQ accordion, mobile nav)
- `hls.js` 1.x — HLS video streaming for hero + footer backgrounds
- `@gsap/react` — React-safe GSAP context + useGSAP hook

**Storage**: N/A — static site, no database

**Testing**: Vite preview + manual browser testing (no unit test framework required for v1)

**Target Platform**: Web — evergreen browsers (Chrome, Firefox, Safari, Edge). Mobile-first responsive.

**Project Type**: Static marketing SPA

**Performance Goals**: LCP < 2.5s, hero text visible < 1s after loading screen exit, 60fps animations

**Constraints**: No SSR, no API, no auth. All external links (Hotmart) are `href` placeholders.
Video placeholder used until client provides HLS URL.

**Scale/Scope**: ~15 React components, ~1500 lines of code. Single page, single bundle.

## Phase 0: Research Decisions

| Decision | Choice | Rationale |
|---|---|---|
| HLS placeholder | Dark CSS gradient fallback | Client video not yet available; gradient ensures no broken state ever ships |
| GSAP vs FM split | GSAP: counter, hero entrance, marquee · FM: section reveals, accordion, AnimatePresence | GSAP excels at imperative timeline control; FM excels at declarative reactive animations |
| CSS approach | Tailwind + CSS custom properties in `index.css` | Brand tokens as CSS vars; Tailwind for layout utilities |
| Scroll reveal | Framer Motion `whileInView` + `once: true` + `margin: "-100px"` | Simple, declarative, no ScrollTrigger overhead for basic reveals |
| FAQ accordion | Framer Motion `AnimatePresence` + `motion.div` height animation | Smooth height without measuring; single-open enforced via component state |
| Fonts | Google Fonts via `<link rel=preconnect>` + `@import` in `index.css` | Standard Vite approach, reliable cross-browser |
| Particle system | Vanilla canvas in `useEffect` (ported from reference HTML) | No extra lib; already proven in reference |

## Implementation Phases

### Phase A — Project Scaffold
- Init Vite + React + TypeScript (`npm create vite@latest seenlabs-web -- --template react-ts`)
- Install deps: `tailwindcss postcss autoprefixer gsap @gsap/react framer-motion hls.js`
- Configure `tailwind.config.ts` with brand CSS custom property tokens
- Write `index.css`: CSS vars (purple, dark backgrounds), keyframes (pulse, doorBreath, scrollDown), base body styles
- Set up Google Fonts (Sora + Manrope) in `index.html`
- Populate all `data/` files with content from `seen-labs.html`
- Write `lib/constants.ts`

### Phase B — Loading Screen
- Build `LoadingScreen.tsx`: counter, cycling words with AnimatePresence, progress bar, entrance/exit animations
- Wire into `App.tsx` with `isLoading` state + AnimatePresence

### Phase C — Navbar + Mobile Nav
- Build `Navbar.tsx`: floating pill, logo dot pulse, nav links, "Start The Path" CTA, scroll shadow via `useScrollNavbar`
- Build `MobileNav.tsx`: full-screen overlay, hamburger toggle, AnimatePresence slide

### Phase D — Hero
- Build `useHLS.ts` hook
- Build `HLSVideo.tsx`: hls.js + native fallback + gradient fallback
- Build `ParticleCanvas.tsx`: canvas particle system (ported from reference HTML JS)
- Build `DoorVisual.tsx`: door frame, glow, light, knob, silhouette SVG
- Build `HeroSection.tsx`: video bg, door, particles, GSAP timeline entrance, cycling role text, CTAs, scroll indicator

### Phase E — Content Sections
- Build all 10 content sections + their UI subcomponents (ProblemCard, PillarCard, PathStep, PricingCard, CaseCard, TeamCard, FAQItem)
- Each section uses `motion.div` whileInView for reveal
- FAQ uses AnimatePresence for accordion

### Phase F — Final CTA + Footer + Marquee
- Build `Marquee.tsx` with GSAP `xPercent` infinite loop
- Build `FinalCTASection.tsx`: second HLS video (heavy overlay), large door visual, marquee, footer bar

### Phase G — Polish + QA
- Wire `useReducedMotion` — skip/simplify all animations
- Responsive QA: 1200px, 900px, 600px, 390px
- Copy audit: verify all text matches reference HTML
- Placeholder audit: video fallback, image placeholders, `#` CTA links flagged
- Performance check: LCP, font load, bundle size
